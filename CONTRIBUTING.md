# Contributing

Thanks for improving the Dev-System-Awesome template. This repository is a
baseline for future projects, so changes should be broadly reusable.

## Contribution Standards

- Keep content general enough for many repositories.
- Prefer docs and workflow patterns that work for public and private repos.
- Include validation commands in pull requests.
- Use conventional commit subjects where practical.
- Update documentation when changing repository behavior.

## Branching

- `main` is the protected template branch.
- Use short-lived feature branches.
- Open pull requests for all changes.
- Delete merged feature branches after publication.

## Pull Request Expectations

A pull request should explain:

- why the change belongs in the template
- what reusable behavior it adds or changes
- how it was validated
- what risks or follow-up work remain

## Review Bar

Template changes should not merge unless:

- documentation builds cleanly
- workflows use least-privilege permissions
- security-sensitive behavior is documented
- project-specific content has been excluded
