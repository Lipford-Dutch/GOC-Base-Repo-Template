# Support

This template provides starting standards for new repositories. Product-specific
support information belongs in the repository created from the template.

## Getting Help

Use GitHub issues for:

- template gaps
- broken workflows
- unclear documentation
- missing kickoff assets
- reusable process improvements

Use a pull request for proposed fixes when the change is clear.

## Escalation

Security issues should follow `SECURITY.md`. Operational incidents should use the
runbook format in `templates/runbook.md`.
