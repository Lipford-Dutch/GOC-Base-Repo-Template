# Release Checklist

## Release

- Version:
- Date:
- Owner:
- Approvers:

## Pre-Release

- [ ] CI is green.
- [ ] Documentation is updated.
- [ ] Release notes are complete.
- [ ] Security findings are resolved or accepted.
- [ ] Rollback path is documented.
- [ ] Known limitations are documented.
- [ ] Artifacts build reproducibly.

## Validation

- [ ] Unit tests pass.
- [ ] Integration tests pass.
- [ ] Docs build passes.
- [ ] Dependency scan reviewed.
- [ ] Smoke test completed.

## Promotion

- [ ] Development validation complete.
- [ ] Staging validation complete.
- [ ] Production approval complete.

## Post-Release

- [ ] Release tag created.
- [ ] Release notes published.
- [ ] Monitoring checked.
- [ ] Issues or follow-ups created.
